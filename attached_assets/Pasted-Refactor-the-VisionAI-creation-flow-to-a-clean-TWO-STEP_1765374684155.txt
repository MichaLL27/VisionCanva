Refactor the VisionAI creation flow to a clean TWO-STEP AI PIPELINE.
The user must NEVER see or edit the image prompt.

CURRENT PROBLEM:
- The user sees the raw AI prompt.
- The flow feels broken and confusing.
- Prompt generation and image generation are mixed inside UI.

REQUIRED FINAL FLOW:

1) USER INPUT (FRONTEND)
The user provides ONLY:
- focusAreas: array of selected focus areas (strings)
- visionText: free text in Hebrew describing their dream life
- style (optional): "modern", "colorful", "spiritual"

They click ONE button only:
"צרו לוח חזון"

--------------------------------
2) STEP 1 – TEXT → IMAGE PROMPT (BACKEND ONLY)

Create a backend function:
generateImagePrompt()

Input:
{
  focusAreas: string[],
  visionText: string,
  style: string
}

Send this to OpenAI TEXT MODEL (ChatGPT):
- The prompt must:
  - Be generated in ENGLISH
  - Be cinematic
  - Be detailed
  - Be suitable for high-quality vision board image
  - Include lighting, mood, environment, composition
  - Include references to goals ONLY in visual form

Return from this function ONLY:
{
  imagePrompt: string
}

❗ This prompt is NEVER exposed to the frontend UI.

--------------------------------
3) STEP 2 – IMAGE GENERATION (BACKEND ONLY)

Create a second backend function:
generateVisionBoardImage()

Input:
{
  imagePrompt: string
}

Send ONLY the AI-generated imagePrompt (not the user text) to the OpenAI IMAGE MODEL.

Return:
{
  imageUrl: string
}

--------------------------------
4) FINAL DATA RETURN TO FRONTEND

Return to the frontend ONLY:

{
  imageUrl: string,
  visionSummaryHebrew: string
}

Where:
- visionSummaryHebrew is a short clean summary generated by the TEXT model in Hebrew
- The raw prompt is NEVER returned to the client.

--------------------------------
5) FRONTEND FLOW UPDATE

After the user clicks "צרו לוח חזון":

- Navigate immediately to loading screen:
  "מייצרים את לוח החזון שלך..."

- Call:
  POST /api/generateVisionBoard

- Wait for imageUrl
- Then navigate to the RESULT screen and display:
  ✅ The image
  ✅ The Hebrew vision summary
  ✅ Buttons:
     - שלחו להדפסה
     - צרו לוח נוסף
     - הורידו כ-PDF

--------------------------------
6) VARIATION LOGIC

When user clicks "צרו לוח נוסף":

- Reuse the SAME imagePrompt
- Append internally: "new variation, different composition"
- Call ONLY generateVisionBoardImage()
- Do NOT regenerate a new prompt

--------------------------------
7) REMOVE THESE COMPLETELY FROM UI:

❌ Any visible AI prompt screen
❌ Any prompt preview textarea
❌ Any "צרו פרומפט חדש"
❌ Any English prompt text shown to users

--------------------------------
8) VALIDATION & SAFETY

- If visionText length < 10 characters → block and show:
  "נא להזין תיאור ברור יותר של החזון שלך"

- Sanitize user input:
  - Strip HTML
  - Strip random spam characters
  - Normalize whitespace
